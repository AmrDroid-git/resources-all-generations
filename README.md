# Resources All Generations

A simple static website for sharing student resources by university/category. The project loads resources from JSON files, displays them in a clean searchable interface, and lets users download selected links as a `.txt` file.

The website is made to be easy to maintain: to add a new university or resource category, you only add a new JSON file inside `data/link/`. No manual JavaScript editing is needed.

## Main Features

- Static website, no database required.
- Works on Netlify and Vercel.
- Multilingual interface: French, English, and Arabic.
- French is the default language.
- Light and dark mode support.
- Multiple visual styles: normal, classic, bold, hacking, modern, academic, minimal, and glass.
- Style selector in the navbar.
- Resource search by name, description, owner, link, university, and type.
- Type filters for files, folders, websites, and all resources.
- Download page that converts selected JSON resources into a `.txt` file.
- Collaboration page linked to a Google Form.
- About page loaded from `data/about.json`.
- Resource file discovery through an auto-generated `_files.json` file.

## How It Works

The website does not hard-code university names in JavaScript. Instead, it reads the list of JSON files inside `data/link/` from this generated file:

```txt
data/link/_files.json
```

The file is generated by:

```bash
node scripts/generate-resource-index.js
```

For example:

```txt
data/link/ensi.json  ->  ENSI
data/link/iseah.json ->  ISEAH
data/link/my-university.json -> MY UNIVERSITY
```

This keeps the project easy to update. You can add, remove, or rename a resource category by changing files inside `data/link/`.

## Run Locally

Go inside the website folder:

```bash
cd website
```

Generate the resource index:

```bash
npm run build
```

Start a local static server:

```bash
npm start
```

Open the website:

```txt
http://localhost:8000/home/
```

Do not open the HTML files directly with `file://`, because the browser may block JSON loading. Always use a local server.

## Available Scripts

## Configuration

Main project settings are in:

```txt
data/config.json
```

Example:

```json
{
  "last_modification_date": "20/06/2026",
  "share_resources_link": "https://docs.google.com/forms/...",
  "resources_hub_link": "https://ressources-hub.netlify.app/",
  "defaults": {
    "language": "fr",
    "theme": "light",
    "style": "classic",
    "default_university": "ensi"
  },
  "resources_directory": "/data/link/"
}
```

Important fields:

| Field                         | Purpose                                                      |
| ----------------------------- | ------------------------------------------------------------ |
| `last_modification_date`      | Date displayed in the navbar.                                |
| `share_resources_link`        | Google Form link used in the collaboration page.             |
| `defaults.language`           | Default interface language.                                  |
| `defaults.theme`              | Default theme: `light` or `dark`.                            |
| `defaults.style`              | Default visual style.                                        |
| `defaults.default_university` | University/category loaded by default on the resources page. |
| `resources_directory`         | Folder containing the resource JSON files.                   |

## Resource JSON Format

Each resource category is a JSON file inside:

```txt
data/link/
```

Example:

```json
[
  {
    "type": "folder",
    "name": "1st year resources",
    "description": "Courses, TDs, exams and useful documents.",
    "link": "https://drive.google.com/drive/folders/...",
    "owner": "community",
    "color": "rgb(255, 160, 71)"
  }
]
```

Recommended fields:

| Field         |    Required | Description                                                 |
| ------------- | ----------: | ----------------------------------------------------------- |
| `type`        |         Yes | Resource type: `file`, `folder`, or `website`.              |
| `name`        |         Yes | Resource title shown on the card.                           |
| `description` | Recommended | Short explanation of the resource.                          |
| `link`        |         Yes | URL to open the resource.                                   |
| `owner`       | Recommended | Owner or contributor name. Use `community` if unknown.      |
| `color`       |    Optional | Custom card accent color. This is ignored in TXT downloads. |

## Add a New University or Category

Create a new JSON file inside `data/link/`:

```txt
data/link/new-university.json
```

Add resources using the resource JSON format.

Then regenerate the resource index:

```bash
npm run build
```

The website will automatically show the new category as:

```txt
NEW UNIVERSITY
```

## Rename a University or Category

Rename the JSON file.

Example:

```txt
data/link/iseah.json
```

becomes:

```txt
data/link/iseah-mahdia.json
```

Then run:

```bash
npm run build
```

The displayed name changes automatically from `ISEAH` to `ISEAH MAHDIA`.

## Edit Styles

All CSS is in:

```txt
assets/css/styles.css
```

The current style system uses CSS variables with:

```css
body[data-style="classic"][data-theme="light"] { ... }
body[data-style="classic"][data-theme="dark"] { ... }
```

To add a new style:

1. Add the style to `appearance.styles` in `data/config.json`.
2. Add matching CSS variable blocks in `assets/css/styles.css`.
3. The style will appear automatically in the navbar selector.

## Deployment on Netlify

Recommended Netlify settings if your repository root contains the `website/` folder:

```txt
Base directory: website
Build command: npm run build
Publish directory: .
```

The included `netlify.toml` already contains:

```toml
[build]
  publish = "."
  command = "node scripts/generate-resource-index.js"
```

So if Netlify uses `website/` as the base directory, it will generate `_files.json` automatically before publishing.

## Important Notes

- Do not manually edit `data/link/_files.json` unless you know what you are doing.
- Run `npm run build` after adding, deleting, or renaming files in `data/link/`.
- The browser cannot reliably list files inside a folder on production hosting, so `_files.json` is needed.
- The website is static, so all resource data is public.
- The `color` field changes the visual card accent only and is not included in exported TXT files.

## Troubleshooting

### Resources are not showing

Run:

```bash
npm run build
```

Then check that this file exists:

```txt
data/link/_files.json
```

### New university does not appear

Make sure the file is inside `data/link/` and ends with `.json`.

Valid:

```txt
data/link/ensi.json
```

Invalid:

```txt
data/ensi.json
data/link/ensi.txt
```

Then run:

```bash
npm run build
```

### Website works locally but not after deployment

Check these deployment settings:

```txt
Build command: npm run build
Output/Publish directory: .
Root/Base directory: website
```

### JSON loading error locally

Do not open pages directly from the file system. Use:

```bash
npm start
```

Then open:

```txt
http://localhost:8000/home/
```
